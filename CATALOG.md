## 17. Detail-Page UX Patterns (from reference sites)

Synthesised from TourRadar (india), Pickyourtrail, Expedia, Tripadvisor, JustWravel, Exoticamp, Visit Abu Dhabi. The MHB destination detail page follows this section order:

1. **Breadcrumb** — Home › Destinations › {Destination}
2. **Hero** — full-bleed destination image, name, one-line tagline, star rating + review count, "from ₹X" price, primary CTA. Trust signals (4.6★, 10k+ reviews, 24/7 support).
3. **Quick-info bar** — duration · best time to visit · ideal for · starting price (sticky-ish strip).
4. **Overview** — 2–3 paragraph "about the destination".
5. **Packages list** — cards each showing: image, type badge (Couple/Family/Group), duration, route ("Singapore 3N +1"), rating, from-price + strikethrough, "View details" CTA.
6. **Highlights / things to do** — icon + label grid.
7. **Sample itinerary** — day-by-day accordion (Day 1…N).
8. **Inclusions & exclusions** — two-column check / cross lists.
9. **Why book with MHB** — reuse 4 USPs (24×7, best price, dedicated advisor, 100% satisfaction).
10. **Reviews** — traveler cards w/ rating + photo (reuse Moments/Stories context).
11. **Best time to visit** — seasonal guidance.
12. **FAQ** — accordion.
13. **Enquiry CTA band** — name + phone + email + "Get a callback", plus WhatsApp/phone shortcuts.
14. **Footer** — shared site footer.

**Card pattern (listing):** rating & review count · hero image w/ optional map overlay · type badge · short blurb · duration / destinations / group-size / languages · operator/brand · original + discounted price · "View tour" + secondary action.

**Trust signals seen across sites:** verified reviews count, Trustpilot/Google rating, 24/7 support, tourism-board partners, "best price guarantee", secure payment / EMI, free cancellation windows.
